Overview

Research Sessions

Conducted 3 research sessions analyzing external Bible APIs. Conclusion: Build on our own D1/Vectorize stack (BTB interlinear data is superior), use external APIs as secondary sync sources.

gslogimaker Plugin

Adopted the gslogimaker plugin reference pattern for dynamic code-block rendering and layered toggles for translations and morphology.

Pipeline Architecture

Recommendation: Leverage AO Lab for breadth (1,000+ translations) and our BTB (D1/Vectorize) stack for depth (interlinear & morphology).

API Candidates

✅ AO Lab (bible.helloao.org)

Findings: 1,000+ translations, no auth, no limits. Best bulk sync source. Includes commentary and audio. ★

✅ BibleQL (bibleql.dev)

Findings: GraphQL, 43 translations, free/open-source. Ideal for Roundtable sermon system.

✅ bible-api.com

Findings: KJV+WEB, clean JSON, no key. Quick lookups only.

✅ getbible.net

Findings: 40+ translations, no key. Multi-translation candidate.

🔜 api.bible (Life.Church)

Findings: 50+ translations, free key. Future addition.

🔜 ESV API

Findings: ESV only, free key. Niche.

❌ bolls.life

Findings: React SPA, no REST API, Cloudflare blocks. Dead end.
