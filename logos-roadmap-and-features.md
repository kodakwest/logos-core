# LogOS Ecosystem Feature Roadmap & Deep Dive

This document outlines the strategic roadmap for the LogOS ecosystem (Core, Shepherd, and Roundtable) with a specific deep dive into the LogOS Roundtable app, focusing on the Sermon Graph, collaborative workbenches, and interactivity.

---

## Part 1: High-Level Ecosystem Roadmap

### Phase 1: Foundation (LogOS Core MVP)
*Focus: Establish the "Operating System for Truth" with robust data ingestion and core AI capabilities.*
- **Data Layer:** Full ingestion of BTB interlinear texts (OT & NT) into D1.
- **Search Engine:** Vectorize-powered semantic search and keyword search.
- **AI Integration:** Llama-4 powered Greek parsing and basic Q&A via the AI Chat sidebar.
- **UI:** Single-page application with split-pane view (Search/History, Verse Display, AI Analysis Panel).

### Phase 2: The Extended Family (Shepherd & Roundtable MVP)
*Focus: Branching out to distinct user personas (Children/Parents and Pastoral Teams).*
- **LogOS Shepherd (Beta):**
  - "Parent Companion Mode" offering guided theological explanations for kids.
  - Interactive "Stepping Stones" UI for sequential learning.
  - Basic memory verse curriculum.
- **LogOS Roundtable (Beta):**
  - Introduction of the spatial **Sermon Graph** (2D canvas).
  - Basic node creation (Text Nodes, Note Nodes).
  - Manual connection lines between nodes.

### Phase 3: Collaborative Intelligence (The Workbench Era)
*Focus: Deep connectivity, multiplayer collaboration, and advanced AI agency.*
- **LogOS Roundtable (V1):** Real-time multiplayer canvas (via WebSockets/Cloudflare Durable Objects).
- **Cross-App Sync:** Discoveries made in Core can be sent directly to a Roundtable project.
- **AI Agentic Workflows:** AI proactively suggests connections between scripture and current events or historical contexts.
- **LogOS Shepherd:** Voice interaction (text-to-speech AI guides) and gamified progression.

### Phase 4: Platform Scale & Ecosystem Integration
*Focus: Making LogOS an extensible platform.*
- **Public API:** Allowing third-party integrations.
- **Community Graph:** Publishing Roundtable graphs for public learning (e.g., a shared public graph of Romans).
- **Custom AI Models:** Fine-tuning embedding models on specific theological libraries (Spurgeon, Calvin, Aquinas).

## Part 2: Deep Dive into LogOS Roundtable

LogOS Roundtable is designed to move pastoral preparation and deep theological study away from linear word processors and into a spatial, interconnected environment.

### 1. The Sermon Graph Concept
The core UI of Roundtable is an infinite 2D canvas where thoughts are treated as interconnected nodes.

**Node Types:**
- **Scripture Nodes:** Direct imports from LogOS Core. Automatically pulls in the text, translation, and underlying Greek/Hebrew morphology.
- **Exegesis Nodes:** The user's own textual analysis and notes attached to a scripture node.
- **Illustration/Application Nodes:** Modern-day stories, historical quotes, or practical applications.
- **AI Insight Nodes:** Automatically generated by the LogOS AI. When a user highlights a connection, the AI might spawn a node suggesting, *"Consider how this links to the Davidic Covenant in 2 Samuel 7."*

**The Graph Engine:**
- **Semantic Edges:** Connections between nodes aren't just visual lines; they have semantic meaning (e.g., "Fulfills", "Contrasts with", "Provides context for").
- **Auto-Clustering:** Using embeddings, the AI can visually group related nodes together on the canvas to help the user identify themes.
- **Pathfinding (The Sermon Arc):** The user defines a "Path" through the graph. This Path represents the linear narrative flow of the actual sermon. The UI allows the user to click through the path to preview the sermon structure.

### 2. Collaborative Tools & Workbenches
Roundtable is built for teams (Lead Pastor, Teaching Pastor, Research Assistant, Worship Leader).

**The Multi-Player Workbench:**
- **Real-Time Presence:** See cursors and active selections of other team members on the canvas (similar to Figma or Miro).
- **Role-Based Views:**
  - *Researcher View:* Focuses on generating Exegesis and Scripture nodes.
  - *Pastor View:* Focuses on arranging nodes into the final "Sermon Arc".
  - *Worship Leader View:* Has read-only access to the graph but can attach "Worship Element Nodes" (songs, liturgy) to specific parts of the sermon arc to ensure thematic cohesion.

**Asynchronous Collaboration:**
- **Threaded Comments:** Users can attach a comment thread to any specific node or connection line.
- **"Needs Review" Status:** A researcher can flag a Greek morphology node for the Lead Pastor to verify.
- **Snapshot History:** The ability to rewind the graph to a previous state (e.g., "What did our outline look like last Tuesday?").

### 3. Interactivity & UX Patterns

**Frictionless Capture:**
- **Command Menu (Cmd/Ctrl + K):** A floating palette accessible anywhere on the canvas to instantly search the Bible or invoke the AI without breaking flow.
- **Drag-and-Drop from Core:** If a user has LogOS Core open in a split screen or side panel, they can physically drag a verse or a Greek lemma onto the Roundtable canvas to spawn a node.

**AI-Assisted Interactivity:**
- **"Expand Context" Action:** Right-clicking a Scripture Node and selecting "Expand Context" prompts the AI to spawn surrounding verses or cross-references as connected nodes.
- **The "Devil's Advocate" Toggle:** A specific AI mode where the user can ask the system to challenge their current exegesis or point out potential counter-arguments or difficult alternative interpretations based on historical theology.

**Export & Delivery:**
- **Linearization Engine:** When the graph is complete, the "Export" function traces the user-defined Sermon Arc and generates a linear Word/PDF document, complete with footnotes for the exegesis and inline Greek text.
- **Presenter Mode:** A distraction-free, high-contrast view of the Sermon Arc optimized for reading on an iPad while preaching on stage, allowing the pastor to tap a node to temporarily reveal the deep exegesis if needed in the moment.
