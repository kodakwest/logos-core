Bible Ecosystem Unification Strategy

Bible AI Search → Shepherd → Roundtable • May 16 2026

Guiding Principles

  1. No deep integration — each app stands on its own. The hub links to them, doesn't absorb them.
  2. Bible knowledge from one source — Bible AI Search D1 DB is the single source of truth for scripture data.
  3. Sermon knowledge from one source (future) — all sermon content eventually comes from a unified source. Plan for it, don't overbuild yet.
  4. Shepherd is ready now — links go live immediately from the hub.
  5. Roundtable comes later — Coming Soon placeholder with vision/roadmap page until ready.

Gap Analysis

No auth on Bible AI Search Every endpoint is public — upload, Workers AI, verse data. CORS is *. Blocks public launch.
AI cost exposure Without auth, anyone can hit /api/ask and burn Workers AI credits.
No hub frontend Bible AI Search has 3 basic views (search, parse, upload). No portal navigation, no landing page.
No Q&A or Explain UI Both /api/ask and /api/explain work but have no frontend interface.
Roundtable not deployed Can't link to something that doesn't exist at a URL. wrangler.toml not committed.
Sermon study section doesn't exist No content model yet. Roundtable will fill this when ready.
No unified sermon knowledge source Future concern — sermon transcripts, guides, series metadata all need a home.

Phase 1: Harden & Hub-ify Bible AI Search

PHASE 1 Bible AI Search becomes the central portal with secure endpoints
  1. 1.1 Auth layer on API endpoints — Protect AI-cost endpoints (/api/ask, /api/explain, /api/upload/*). Public read on verse search data.
  2. 1.2 Redesign frontend as portal — Landing page with section navigation. Keep existing Search + Parse views intact.
  3. 1.3 Add Q&A interface — Frontend UI for POST /api/ask. Ask bar, cited answer display.
  4. 1.4 Add verse deep-dive UI — Frontend UI for POST /api/explain. Verse card with context, morphology, analysis.
  5. 1.5 Kids Ministry section → Shepherd — Hub section with description + direct link to shepherdparentcompanion.pages.dev
  6. 1.6 Sermon Study section → Coming Soon — Hub section with vision description + roadmap page. Links to Roundtable when ready.
  7. 1.7 Responsive portal navigation — Mobile-friendly nav between Bible Study, Kids Ministry, Sermon Study sections.

Phase 2: Complete & Connect Roundtable

PHASE 2 Roundtable is built, deployed, and linked from the hub
  1. 2.1 Finish Roundtable — Markdown export, deeperDive rendering, remaining guides, commit wrangler.toml
  2. 2.2 Deploy to CF Pages — Make it live at a URL
  3. 2.3 Update hub Sermon Study section — Replace Coming Soon with live links to Roundtable
  4. 2.4 Plan unified sermon knowledge source — Design how sermon guides, transcripts, and series data can come from one source
